Setting up Squid (3) Transparent Proxy on Raspberry Pi


Here are the steps on how I setup squid (3) transparent proxy on raspberry pi.

Update the package list

sudo apt-get update

Install squid3 package

sudo apt-get install squid3

I'll be using a different directory for cache and logs to save some space on my sdcard. For steps on how to use and automount flash drive on raspberry pi you can follow this guide:

http://gagsap37.blogspot.com/2013/09/automount-usb-flash-drive-on-raspbian.html

Create the directory where the cache and logs will be saved. Here my mountpoint is /mnt/usbdrive

mkdir -p  /mnt/usbdrive/proxy/cache
mkdir -p  /mnt/usbdrive/proxy/logs

change owner & group of proxy directory to proxy:proxy

cd /mnt/usbdrive
chown -Rv proxy:proxy proxy

Edit the squid3 configuration file "/etc/squid3/squid.conf"

sudo nano /etc/squid3/squid.conf

Here is my squid.conf file and I highlighted the lines that I've change

acl manager proto cache_object
acl localhost src 127.0.0.1/32 ::1
acl to_localhost dst 127.0.0.0/8 0.0.0.0/32 ::1
acl localnet src 192.168.137.0/24 #change this to reflect my network settings usually this is 192.168.0.0
acl SSL_ports port 443
acl Safe_ports port 80          # http
acl Safe_ports port 21          # ftp
acl Safe_ports port 443         # https
acl Safe_ports port 70          # gopher
acl Safe_ports port 210         # wais
acl Safe_ports port 1025-65535  # unregistered ports
acl Safe_ports port 280         # http-mgmt
acl Safe_ports port 488         # gss-http
acl Safe_ports port 591         # filemaker
acl Safe_ports port 777         # multiling http
acl CONNECT method CONNECT
http_access allow manager localhost
http_access deny manager
http_access deny !Safe_ports
http_access deny CONNECT !SSL_ports
http_access allow localhost
http_access allow localnet #tells squid to allow any client on my network to use the proxy service
http_access deny all
http_port 3128 transparent #add the keywork transparent to enable tranparent proxy
 
cache_dir ufs /mnt/usbdrive/proxy/cache 2048 16 256 #points cache_dir to the directory on my flash drive
maximum_object_size 51200 KB #adjust the size of the objects to be cache on the proxy server anything larger will not be cache
access_log /mnt/usbdrive/proxy/logs/access.log squid #points access_log to the directory on my flash drive
cache_store_log /mnt/usbdrive/proxy/logs/store.log #points cache_store_log to the directory on my flash drive
cache_log /mnt/usbdrive/proxy/logs/cache.log #points cache_log to the directory on my flash drive
coredump_dir /mnt/usbdrive/proxy/ #points coredump_dir to the directory on my flash drive
refresh_pattern ^ftp:           1440    20%     10080
refresh_pattern ^gopher:        1440    0%      1440
refresh_pattern -i (/cgi-bin/|\?) 0     0%      0
refresh_pattern .               0       20%     4320
dns_nameservers 192.168.137.1 #points squid to use the caching dns that is setup on my router

Save.

initialize the cache directory

sudo squid3 –z 

start the squid3 server

sudo service squid3 start

Test squid by pointing your browsers proxy setting to the raspberry pi ip address then set the port to 3128.






















Try to browse somewebsite and see if its working.

Once squid is confirmed working, next step is to setup my router to re-route any traffic destined to port 80 to port 3128 of raspberry pi.


For that I add the following rules on iptables of my router (which is running tomato by shibby).

iptables -t nat -A PREROUTING -i br0 -s 192.168.137.1 -d 255.255.255.0 -p tcp --dport 80 -j ACCEPT
iptables -t nat -A PREROUTING -i br0 -s ! 192.168.137.71 -p tcp --dport 80 -j DNAT --to 192.168.137.71:3128
iptables -t nat -I POSTROUTING -o br0 -s 192.168.137.1 -d 192.168.137.71 -p tcp -j SNAT --to 192.168.137.1
iptables -I FORWARD -i br0 -o br0 -s 192.168.137.1 -d 192.168.137.71 -p tcp --dport 3128 -j ACCEPT

Remove the proxy settings on your browser, and try again to browse any website. If all is well you see see the website loads.
